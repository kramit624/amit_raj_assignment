## Quick Start (Local)

### 1. Clone & set up Python environment

```bash
git clone https://github.com/kramit624/amit_raj_assignment.git
cd amit_raj_assigment

python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
```

---

### 2. Run the data pipeline

```bash
# Step 1 — Clean raw CSVs → data/processed/
python scripts/clean_data.py

# Step 2 — Merge + analyse → 4 output CSVs
python scripts/analyze.py
```

Both scripts accept optional flags:
```bash
python scripts/clean_data.py --raw_dir data/raw --out_dir data/processed
python scripts/analyze.py --processed_dir data/processed --raw_dir data/raw
```

**Output files in `data/processed/`:**

| File                      | Contents                                   |
|---------------------------|--------------------------------------------|
| `customers_clean.csv`     | Cleaned customer records                   |
| `orders_clean.csv`        | Cleaned order records                      |
| `monthly_revenue.csv`     | Monthly revenue trend (completed orders)   |
| `top_customers.csv`       | Top 10 customers by spend + churn flag     |
| `category_performance.csv`| Revenue, avg order value, count by category|
| `regional_analysis.csv`   | KPIs grouped by region                     |

---

### 3. Start the backend

```bash
# From project root
uvicorn backend.main:app --reload
```

API runs at **http://localhost:8000**

| Endpoint              | Description                    |
|-----------------------|--------------------------------|
| `GET /health`         | Liveness check → `{status:ok}` |
| `GET /api/revenue`    | Monthly revenue trend          |
| `GET /api/top-customers` | Top 10 customers + churn    |
| `GET /api/categories` | Category performance           |
| `GET /api/regions`    | Regional analysis              |

---

### 4. Start the frontend

```bash
cd frontend
npm install
npm run dev
```

Dashboard runs at **http://localhost:5173**

## Run Tests

```bash
# From project root
pytest tests/ -v

# Single file
pytest tests/test_clean_data.py -v

# With coverage
pytest tests/ -v --tb=short
```

The test suite covers:

- `is_valid_email` — 7 cases (valid, missing @, missing dot, null, empty, NaN)
- `parse_dates` — 8 cases (all 4 formats, bad values, None, mixed series)
- `clean_customers` — 11 cases (dedup, whitespace, email flags, region fill, date parsing)
- `clean_orders` — 13 cases (drop unrecoverable, fill amounts, status vocab, date formats)

## Docker (Bonus)

### Build and run with Docker Compose

```bash
# Build image + run pipeline + start API
docker-compose up --build

# Stop
docker-compose down
```

The `data/processed/` directory is bind-mounted, so output CSVs persist on your machine between container restarts.

### Manual Docker

```bash
docker build -t sales-dashboard-backend .
docker run -p 8000:8000 -v $(pwd)/data:/app/data sales-dashboard-backend
```

## Design Decisions & Assumptions

- `products.csv` is treated as a clean reference file (no cleaning step needed).
- Churn is defined as **no completed order in the last 90 days** relative to the latest order date in the dataset.
- Orders with unparseable dates have `order_date = NaT` and are excluded from `order_year_month` grouping.
- Missing `amount` values are filled with the **median grouped by product** (global median used as fallback if all values for a product are null).
- Status normalisation uses a priority map: exact match → direct map → partial match → default `"pending"`.
- All file paths use `pathlib.Path(__file__).resolve().parent` — no hardcoded absolute paths anywhere.
